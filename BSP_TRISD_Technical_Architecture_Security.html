3. Multi-Layer Security Architecture

üåê Layer 1: Network Security

Virtual Private Cloud (VPC)
Isolated network with private subnets for databases and microservices. Public subnets only for load balancer. No direct internet access to application servers.
Security Groups & NACLs
Firewall rules at instance and subnet levels. Least-privilege principle - only required ports open (443 HTTPS, internal service ports). DDoS protection via AWS Shield.
Web Application Firewall (WAF)
Protects against OWASP Top 10: SQL injection, XSS, CSRF. Custom rules for rate limiting (max 100 req/min per IP). Geo-blocking for high-risk countries.
DDoS Mitigation
AWS Shield Standard (free tier) for network/transport layer protection. CloudFront edge locations absorb traffic spikes. Auto-scaling to handle legitimate surges.

üîë Layer 2: Authentication & Authorization

Multi-Factor Authentication (MFA)
Mandatory for transactions >‚Ç±10,000. SMS OTP (Twilio) or authenticator app (TOTP). Biometric authentication on mobile (fingerprint, Face ID). Device binding for trusted devices.
JWT Token Management
Short-lived access tokens (15 min expiry). Refresh tokens (7 days) stored in HttpOnly cookies. Token blacklist in Redis for immediate revocation. RS256 algorithm (asymmetric signing).
Role-Based Access Control (RBAC)
5 roles: Consumer, Provider, Admin, Mediator, Super Admin. Permissions matrix controls API endpoint access. Principle of least privilege - users only access what they need.
Session Management
Redis-based session store with 30-min timeout. Concurrent session limit (max 3 devices). Automatic logout on password change. IP address + user-agent validation.

üîí Layer 3: Data Encryption

Encryption at Rest
Database: AES-256 encryption for RDS volumes (PostgreSQL, MongoDB)
Files: S3 bucket encryption (SSE-S3)
Backups: Encrypted snapshots
Sensitive Fields: Additional application-level encryption for PII (government IDs, bank account numbers)
Encryption in Transit
HTTPS Only: TLS 1.3 for all client-server communication. No HTTP allowed (301 redirect to HTTPS).
Internal Services: TLS between microservices
Database Connections: Encrypted PostgreSQL connections
Certificate: AWS Certificate Manager (auto-renewal)
Key Management
AWS Secrets Manager for API keys, database passwords. AWS KMS (Key Management Service) for envelope encryption. Automatic key rotation every 90 days. Keys never exposed in code or logs.
Payment Data Security
PCI-DSS Compliance: Never store credit card data on platform. Tokenization via PayMongo (PCI-DSS Level 1). Only tokens stored in database. E-wallet credentials encrypted at application level.

üõ°Ô∏è Layer 4: Application Security

Input Validation & Sanitization
All inputs validated using Zod schemas. SQL injection prevention via parameterized queries (ORM). XSS protection via output encoding. File upload restrictions (type, size, content validation).
API Rate Limiting
Global: 100 requests/min per IP
Authentication: 5 login attempts/15 min (lockout after)
Transaction: 10 transactions/hour per user
Wallet Top-Up: 5 top-ups/day per user
Rate limits enforced in Redis with sliding window algorithm.
CSRF Protection
Anti-CSRF tokens for all state-changing requests. SameSite cookie attribute (Strict). Origin header validation. Double-submit cookie pattern for API requests.
Security Headers
Content-Security-Policy: Prevents XSS via inline scripts
X-Frame-Options: DENY (clickjacking protection)
Strict-Transport-Security: Force HTTPS
X-Content-Type-Options: nosniff

All critical services deployed across 3 Availability Zones in AWS Singapore region. Automatic failover if one AZ fails. Database replicas in multiple AZs. Load balancer distributes traffic across zones.

Auto-Scaling
Horizontal scaling based on CPU (>70%), memory (>80%), or request rate. Min 2 instances per service (redundancy). Max 20 instances (cost control). Scale-up in 2 min, scale-down in 5 min.
