Legend & Key Definitions

Trust Account: BSP-compliant segregated account for user funds
Escrow: Locked funds until service completion verification
Wallet Balance: User-owned funds available for transactions
Operating Account: Platform revenue and expenses only
Settlement Account: Provider payout processing account
Reserve Buffer: 10% liquidity cushion for withdrawals

Critical BSP Compliance Notes

  • Complete Fund Segregation: User funds in Trust Account never commingled with platform operating funds
  • Daily Reconciliation: Automated daily balance verification with manual audit escalation if variance detected
  • Escrow Protection: 100% of service payments go through escrow mechanism - no direct P2P transfers
  • Insolvency Protection: Even if platform fails, user funds are protected in separate trust account
  • Reserve Requirements: 10% buffer maintained above user balances + escrows for liquidity
  • Transparency: Users can view transaction history and wallet balance in real-time
  • Withdrawal Rights: Users can withdraw available wallet balance anytime (not locked in platform)
  • Audit Trail: Every peso movement tracked with timestamp, user ID, transaction ID for BSP reporting
